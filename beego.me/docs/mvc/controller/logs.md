<h1>Logging</h1>

<p>We have already spoken about how Beego is based on several independent modules. Beego uses the logs module to handle logging. Beego already has a variable <code>BeeLogger</code> which is a <code>logs.BeeLogger</code> type and initialized as <code>console</code> which will output to <code>console</code>.</p>

<h2>Basic usage</h2>

<p>In our Beego application we log as below:</p>

<pre><code>beego.Debug(&quot;this is debug&quot;)
beego.Info(&quot;this is info&quot;)
beego.Notice(&quot;this is notice&quot;)
beego.Warn(&quot;this is warn&quot;)
beego.Error(&quot;this is error&quot;)
beego.Critical(&quot;this is critical&quot;)
beego.Alert(&quot;this is alert&quot;)
beego.Emergency(&quot;this is emergency&quot;)
</code></pre>

<h2>Configure output</h2>

<p>Usually we want to output information into a log file and it&rsquo;s easy to do that:</p>

<pre><code>beego.SetLogger(&quot;file&quot;, `{&quot;filename&quot;:&quot;logs/test.log&quot;}`)
</code></pre>

<p>For more usage of logs, see <a href="../../module/logs.md">logs</a>.<br />
​<br />
After the above setting has been made, logs will output to both console and file. If you only want to output to file, you need to remove console like this:</p>

<pre><code>beego.BeeLogger.DelLogger(&quot;console&quot;)    
</code></pre>

<h2>Configure logging level</h2>

<p>As we saw above, there are 8 different logging levels:</p>

<pre><code>LevelDebug
LevelInformational
LevelNotice
LevelWarning
LevelError
LevelCritical
LevelAlert
LevelEmergency
</code></pre>

<p>The logging level goes from debug to emergency. It will output all by default. We can set the different logging levels on different servers:</p>

<pre><code>beego.SetLevel(beego.LevelInformational)
</code></pre>

<h2>output file and line number</h2>

<p>Log output does not include call file by default, if you&rsquo;re expecting output calls&rsquo; file name and line number, set as follows</p>

<pre><code>beego.SetLogFuncCall(true)
</code></pre>

<p>Default is false, i.e. not to log file names and line numbers.</p>

<h2>Example</h2>

<pre lang="go"><code>func internalCalculationFunc(x, y int) (result int, err error) {
    beego.Debug(&quot;calculating z. x:&quot;,x,&quot; y:&quot;,y)
    z := y
    switch {
    case x == 3:
        beego.Trace(&quot;x == 3&quot;)
        panic(&quot;Failure.&quot;)
    case y == 1:
        beego.Trace(&quot;y == 1&quot;)
        return 0, errors.New(&quot;Error!&quot;)
    case y == 2:
        beego.Trace(&quot;y == 2&quot;)
        z = x
    default:
        beego.Trace(&quot;default&quot;)
        z += x
    }
    retVal := z-3
    beego.Debug(&quot;Returning &quot;, retVal)
    
    return retVal, nil
}   

func processInput(input inputData) {
    defer func() {
        if r := recover(); r != nil {
            beego.Error(&quot;Unexpected error occurred: &quot;, r)
            outputs &lt;- outputData{result : 0, error : true}
        }
    }()
    beego.Info(&quot;Received input signal. x:&quot;,input.x,&quot; y:&quot;, input.y)
    
    res, err := internalCalculationFunc(input.x, input.y)
    if err != nil {
        beego.Warn(&quot;Error in calculation:&quot;, err.Error())
    }
    
    beego.Info(&quot;Returning result: &quot;,res,&quot; error: &quot;,err)
    outputs &lt;- outputData{result : res, error : err != nil}
}

func main() {
    inputs = make(chan inputData)
    outputs = make(chan outputData)
    criticalChan = make(chan int)
    beego.Info(&quot;App started.&quot;)
    
    go consumeResults(outputs)
    beego.Info(&quot;Started receiving results.&quot;)
    
    go generateInputs(inputs)
    beego.Info(&quot;Started sending signals.&quot;)
    
    for {
        select {
            case input := &lt;- inputs:
                processInput(input)
            case &lt;- criticalChan:
                beego.Critical(&quot;Caught value from criticalChan: Go shut down.&quot;)
                panic(&quot;Shut down due to critical fault.&quot;)
        }   
    }
}
