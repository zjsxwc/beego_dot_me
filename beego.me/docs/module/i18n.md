<h1>Introduction to i18n</h1>

<p>This module is mainly used for i18n of sites and applications, which provides multiple-language options to users, improve user experience. Sites like <a href="http://gowalker.org">Go Walker</a> and <a href="http://beego.me">Beego.me</a> use this module to implement Chinese and English user interfaces.</p>

<p>You can use the following command to install this module:</p>

<pre><code>go get github.com/beego/i18n
</code></pre>

<h2>i18n Usage</h2>

<p>First of all, you have to import this package:</p>

<pre><code>import (
    &quot;github.com/beego/i18n&quot;
)
</code></pre>

<p>The format of locale files is very much like the INI format configuration file, which is basically key-value pairs. But this module has some improvements. Every language corresponds to a locale file, for example, under <code>conf</code> folder of beego.me, there are two files called <code>locale_en-US.ini</code> and <code>locale_zh-CN.ini</code>.</p>

<p>The name and extensions of locale files can be anything, but we strongly recommend you to follow the style of beego.me.</p>

<h2>Minimal example</h2>

<p>Here are two simplest locale file examples:</p>

<p>File <code>locale_en-US.ini</code>:</p>

<pre><code>hi = hello
bye = goodbye
</code></pre>

<p>File <code>locale_zh-CN.ini</code>:</p>

<pre><code>hi = 您好
bye = 再见
</code></pre>

<h3>Use in controller</h3>

<p>For every request, Beego uses individual goroutines to handle the request; therefore, you can embed an <code>i18n.Locale</code> as an anonymous field to process locale operations of the current request. This requires that you understand the idea of <code>baseController</code> and <code>Prepare</code> method. See source file <code>routers/router.go</code> of beego.me for more details.</p>

<p>After accepting the request, use the <code>Prepare</code> method of <code>baseController</code> to do language operations, which you only need to write the same code once and use it in all the upper level controllers.</p>

<h4>Register locale files</h4>

<p>The following code is from beego.me source file <code>routers/init.go</code>:</p>

<pre lang="go"><code>// Initialized language type list.
langs := strings.Split(beego.AppConfig.String(&quot;lang::types&quot;), &quot;|&quot;)
names := strings.Split(beego.AppConfig.String(&quot;lang::names&quot;), &quot;|&quot;)
langTypes = make([]*langType, 0, len(langs))
for i, v := range langs {
    langTypes = append(langTypes, &amp;langType{
        Lang: v,
        Name: names[i],
    })
}

for _, lang := range langs {
    beego.Trace(&quot;Loading language: &quot; + lang)
    if err := i18n.SetMessage(lang, &quot;conf/&quot;+&quot;locale_&quot;+lang+&quot;.ini&quot;); err != nil {
        beego.Error(&quot;Fail to set message file: &quot; + err.Error())
        return
    }
}
</code></pre>

<p>In this piece of code, we get languages that we want to support in the configuration file, in this case, we have <code>en-US</code> and <code>zh-CN</code>. Then we initialize a slice for users to change language option(not discussed here). Finally, we call the <code>i18n.SetMessage</code> function in a loop to load all the locale files. Here you can see why we recommend the you use the naming conventions of beego.me for locale files.</p>

<h4>Initialize controller language</h4>

<p>The following code is from the beego.me source file <code>routers/router.go</code>, which decides on which user language option to use in the following order: 1: URL specified 2: Cookies and 3: browser <code>Accept-Language</code>.</p>

<pre lang="go"><code>// setLangVer sets site language version.
func (this *baseRouter) setLangVer() bool {
    isNeedRedir := false
    hasCookie := false

    // 1. Check URL arguments.
    lang := this.Input().Get(&quot;lang&quot;)

    // 2. Get language information from cookies.
    if len(lang) == 0 {
        lang = this.Ctx.GetCookie(&quot;lang&quot;)
        hasCookie = true
    } else {
        isNeedRedir = true
    }

    // Check again in case someone modify on purpose.
    if !i18n.IsExist(lang) {
        lang = &quot;&quot;
        isNeedRedir = false
        hasCookie = false
    }

    // 3. Get language information from 'Accept-Language'.
    if len(lang) == 0 {
        al := this.Ctx.Request.Header.Get(&quot;Accept-Language&quot;)
        if len(al) &gt; 4 {
            al = al[:5] // Only compare first 5 letters.
            if i18n.IsExist(al) {
                lang = al
            }
        }
    }

    // 4. Default language is English.
    if len(lang) == 0 {
        lang = &quot;en-US&quot;
        isNeedRedir = false
    }

    curLang := langType{
        Lang: lang,
    }

    // Save language information in cookies.
    if !hasCookie {
        this.Ctx.SetCookie(&quot;lang&quot;, curLang.Lang, 1&lt;&lt;31-1, &quot;/&quot;)
    }

    restLangs := make([]*langType, 0, len(langTypes)-1)
    for _, v := range langTypes {
        if lang != v.Lang {
            restLangs = append(restLangs, v)
        } else {
            curLang.Name = v.Name
        }
    }

    // Set language properties.
    this.Lang = lang
    this.Data[&quot;Lang&quot;] = curLang.Lang
    this.Data[&quot;CurLang&quot;] = curLang.Name
    this.Data[&quot;RestLangs&quot;] = restLangs

    return isNeedRedir
}
</code></pre>

<p>The variable <code>isNeedRedir</code> indicates whether user uses URL to specify the language option. To keep the URL clean, beego.me automatically sets the value in cookies and redirect.</p>

<p>The line <code>this.Data[&quot;Lang&quot;] = curLang.Lang</code> sets user language option to template variable  <code>Lang</code> so that we can handle language in template files.</p>

<p>Following two lines:</p>

<pre><code>this.Data[&quot;CurLang&quot;] = curLang.Name
this.Data[&quot;RestLangs&quot;] = restLangs
</code></pre>

<p>For users to change language option, see beego.me source code for more details.</p>

<h4>Handle language in controller</h4>

<p>While the <code>i18n.Locale</code> is an anonymous field to be embedded in <code>baseController</code>, we can use <code>this.Tr(format string, args ...interface{})</code> to handle language in controller.</p>

<h3>Handle language in template</h3>

<p>By passing template variable <code>Lang</code> to indicate language option, you are able to do localization in template. But before that, you need to register a template function.</p>

<p>Following code is from beego.me source file <code>beeweb.go</code>:</p>

<pre><code>beego.AddFuncMap(&quot;i18n&quot;, i18n.Tr)
</code></pre>

<p>After that, do the following with <code>Lang</code> to handle language:</p>

<pre><code>{{i18n .Lang &quot;hi%d&quot; 12}}
</code></pre>

<p>Code above will produce:</p>

<ul>
<li>English <code>en-US</code>：<code>hello12</code><br /></li>
<li>Chinese <code>zh-CN</code>：<code>您好12</code><br /></li>
</ul>

<h2>Section</h2>

<p>For different pages, one key may map to different values. Therefore, i18n module also uses the section feature of INI format configuration to achieve section separation.</p>

<p>For example, the key name is <code>about</code>, and we want to show <code>About</code> in the home page and <code>About Us</code> in about page. Then you can do following:</p>

<p>Content in locale file:</p>

<pre><code>about = About

[about]
about = About Us
</code></pre>

<p>Get <code>about</code> in home page:</p>

<pre><code>{{i18n .Lang &quot;about&quot;}}
</code></pre>

<p>Get <code>about</code> in about page:</p>

<pre><code>{{i18n .Lang &quot;about.about&quot;}}
</code></pre>

<h3>Ambiguity</h3>

<p>Because dot <code>.</code> denotes a section in both <a href="https://github.com/Unknwon/goconfig">INI parser</a> and locale files, when your key name contains <code>.</code> this will cause ambiguity. To avoid ambiguity, you just need to add one more <code>.</code> in front of the key.</p>

<p>For example, the key name is <code>about.</code>, then we can use:</p>

<pre><code>{{i18n .Lang &quot;.about.&quot;}}
</code></pre>

<p>to get the desired result.</p>

<h2>Helper tool</h2>

<p>Module i18n provides a command line helper tool beei18n to simplify the steps in your development. You can install it as follows:</p>

<pre><code>go get github.com/beego/i18n/beei18n
</code></pre>

<h3>Sync locale files</h3>

<p>Command <code>sync</code> allows you use a exist local file as the template to create or sync other locale files:</p>

<pre><code>beei18n sync source_file.ini other1.ini other2.ini
</code></pre>

<p>This command can operate on 1 or more files in one command.</p>

<h2>More information</h2>

<p>If the key does not exist, then i18n will return the key string to caller. For instance, when key name is <code>hi</code> and it does not exist in locale file, it will simply return <code>hi</code> as output.</p>
